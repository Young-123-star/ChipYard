# 水电费抄表计费 — 设计文档

> 第二阶段第五个子项目。日期 2026-06-17。分支 `feat/utility-billing`（从最新 `main` 切出）。

## 1. 目标与范围

为 DMS 增加**水电费抄表计费**：按房间抄录水/电表读数 → 用量×单价算费 → 按房间在住人均摊生成账单。水电账单复用现有 `dms_fee_bill`（加 `bill_type` 区分），自动走既有的缴费/退宿挂账/欠费预览流程。全局单一单价。两步流程：先抄表台账，再按期生成账单。仍沿用 H2 内存库 DEMO，子 Agent 驱动 + TDD。

**验收标准：**
- 管理员可维护水电单价（电 元/度、水 元/吨，全局单行配置）。
- 管理员按账期、按房间录入水/电本期读数；系统自动取上期读数、算用量与金额，存抄表台账；同房同期同表幂等（重复录入更新）。
- 按期一键生成水电账单：对已抄表的房间，把每块表费用按房间当前在住人均摊（余数并入第一张），给每个在住居住人各开一张账单（bill_type 区分电/水）；无在住跳过；同档案同期同类型幂等。
- 水电账单与住宿费账单统一走缴费/作废/欠费预览/退宿挂账，互不串味。
- 后端关键逻辑测试覆盖。

## 2. 数据模型

新增 2 表 + 改 1 表（不动现有住宿费数据）。沿用 `BaseEntity`（created_at/updated_at/deleted_at 逻辑删除）。

### dms_meter_reading（抄表台账）
| 列 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| room_id | BIGINT NOT NULL | 房间 |
| period | VARCHAR(7) NOT NULL | 账期 YYYY-MM |
| meter_type | TINYINT NOT NULL | 1电 / 2水 |
| prev_reading | DECIMAL(12,2) | 上期读数（录入时自动取该房该表上一期读数，首期 0） |
| current_reading | DECIMAL(12,2) NOT NULL | 本期读数 |
| consumption | DECIMAL(12,2) | 用量 = 本期-上期（快照） |
| unit_price | DECIMAL(10,2) | 单价快照 |
| amount | DECIMAL(10,2) | 金额 = 用量×单价（快照） |
| created_at / updated_at / deleted_at | DATETIME | |

幂等键（应用层）：`(room_id, period, meter_type)`。

### dms_utility_rate（水电单价，单行配置）
| 列 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK | 固定一行 id=1 |
| electricity_price | DECIMAL(10,2) | 电价 元/度 |
| water_price | DECIMAL(10,2) | 水价 元/吨 |
| created_at / updated_at / deleted_at | DATETIME | |

### dms_fee_bill（改）
加两列：
- `bill_type TINYINT DEFAULT 1`（1住宿费 / 2电费 / 3水费）——现有住宿费账单默认 1。
- `remark VARCHAR(200)`（水电账单明细描述，如「电费 30度×1.00 ÷2人」）。

### 种子
- `dms_utility_rate` 一行：电 1.00、水 5.00。
- 张三所在 A102（room_id=2）：电表上一期 + 本期、水表上一期 + 本期各一条抄表（账期 2026-06 或演示账期），便于直接生成账单演示。

## 3. 计费与生成流程

### 3.1 抄表录入（按期、按房间、按表）
1. 入参：room_id、period、meter_type、current_reading。
2. 取该房该表**上一期**（period < 本期，按 period 倒序第一条）的 `current_reading` 作 `prev_reading`，无则 0。
3. `consumption = current_reading - prev_reading`；`amount = consumption × 当前单价`（电用 electricity_price，水用 water_price）。
4. 幂等：`(room_id, period, meter_type)` 已存在则更新该条读数并重算，否则新建。

### 3.2 生成水电账单（按期，`@Transactional`，幂等批量）
遍历该账期所有抄表台账记录（`amount >= 0`）：
1. 找该房间当前在住居住人（`checkin_record.status=1 AND room_id`）。无在住 → 跳过，`skipped++`。
2. 按在住人数**均摊** amount：每人 = `amount / 人数` 四舍五入(2 位, HALF_UP)，**余数（总额 - 各份合计）并入第一张账单**，保证合计 = 房间总额。
3. 每个在住居住人开一张 `dms_fee_bill`：bill_type（2电/3水）、period、room_id、checkin_record_id、resident_id、amount=分摊额、status=未缴(1)、remark。
4. 幂等：`(checkin_record_id, period, bill_type)` 已存在则跳过。
返回 `GenerateResultVO { generated, skipped }`。

### 3.3 与现有账单体系的衔接（关键）
- **住宿费 generate 幂等加 bill_type 限定**：现有住宿费按 `(checkin_record_id, period)` 判重；加 bill_type 后必须限定 `bill_type=1`，否则与水电账单串味。`getByRecordAndPeriod` 加 `billType` 参数，住宿费生成传 1。住宿费新建账单显式 `bill_type=1`。
- **退宿挂账自动覆盖水电**：`settleArrearsForRecord` 把该档案所有未缴账单置挂账——天然把住宿费+水电一起挂账，无需改动（顺带的好处）。
- **缴费/作废/欠费预览** 与 bill_type 无关，统一复用。

## 4. 后端改动

### 4.1 新增 meter 相关（归 fee 模块 `module/fee/...`）
- 实体：`MeterReading`、`UtilityRate`
- mapper：`MeterReadingMapper`、`UtilityRateMapper`
- dto：`MeterReadingDTO`（room_id/period/meterType/currentReading）、`MeterQuery`（period/roomId/meterType + 分页）、`UtilityRateDTO`（electricityPrice/waterPrice）、`GenerateBillsDTO` 复用（period）
- vo：`MeterReadingVO`（台账 + roomNumber 回显）
- `MeterService` + `MeterServiceImpl`：
  - `UtilityRate getRate()` / `void updateRate(UtilityRateDTO)`
  - `PageResult<MeterReadingVO> pageReadings(MeterQuery)`
  - `Long saveReading(MeterReadingDTO)`（自动取上期、算用量金额、幂等 upsert）
  - `GenerateResultVO generateUtilityBills(String period)`（`@Transactional`，调 FeeBillService 建单）

### 4.2 FeeBillService / 实体 改
- `FeeBill` 实体 + `FeeBillVO` 加 `billType`（Integer）、`remark`（String）。
- `getByRecordAndPeriod(Long, String, Integer billType)` 加 billType 参数；住宿费 generate 判重传 1、新建账单 `setBillType(1)`。
- 新增建单方法供 MeterService 调用，例如 `Long createUtilityBill(Long recordId, Long residentId, Long roomId, String period, Integer billType, BigDecimal amount, String remark)`（落 status=未缴、billNo、bill_type、remark）。

### 4.3 CheckinService 改
- 加 `java.util.List<CheckinRecord> listActiveRecordsByRoom(Long roomId)`（按房间取在住档案，供均摊取在住人）。

### 4.4 接口（均 JWT，挂 `/api/fee`）
- `GET /api/fee/utility-rate`、`PUT /api/fee/utility-rate`
- `GET /api/fee/meter-readings`（分页筛选）、`POST /api/fee/meter-readings`（录入/更新）
- `POST /api/fee/utility-bills/generate { "period": "YYYY-MM" }`

## 5. 前端

「费用管理」菜单组加一页 **抄表/水电** `views/fee/meter.vue`：
- 顶部单价配置条：电价/水价 输入 + 「保存单价」。
- 抄表录入：选账期 + 楼栋→房间联动，录入电/水本期读数（显示自动带出的上期读数、用量、金额预览）；台账表格（房间/账期/类型/上期/本期/用量/单价/金额）。
- 「生成水电账单」按钮（选账期）→ toast「生成 N 张 / 跳过 M 张」。
- 账单页 `views/fee/bill.vue`：加 `BILL_TYPE` 字典（1住宿费/2电费/3水费）+ 类型列 + 类型筛选；remark 作为类型列 tooltip 或单独「明细」列。
- `api/fee.ts` 加 utility-rate / meter-readings / utility-bills 函数；types 加 `MeterReading`/`UtilityRate`，`FeeBill` 加 `billType`/`remark`。
- 路由 + layout 菜单加「抄表/水电」。

## 6. 测试（TDD + 子 Agent 驱动 + 两阶段审查）

- 单价配置读写（get 默认种子值、update 生效）。
- 抄表 saveReading：自动取上期读数、算用量/金额正确、同房同期同表幂等更新（不新增第二条）。
- generateUtilityBills：按在住人均摊（2 人除不尽时余数并入第一张、合计=房间总额）、无在住跳过、幂等重复生成不重复、bill_type/remark 正确。
- 住宿费 generate 幂等加 bill_type=1：住宿费与水电同房同期互不串味（同档案同期可同时存在住宿费+电费+水费各一张）。
- 退宿挂账回归：confirm 把该档案住宿费+水电未缴一起挂账。
- Controller：抄表录入 / 生成 / 单价 接口鉴权（无 token → 401）。
- 全量回归通过。
