# 退宿欠费结算挂账 — 设计文档（费用 × 退宿打通）

> 第二阶段第四个子项目。日期 2026-06-16。分支 `feat/checkout-arrears`（从合并了费用的最新 `main` 切出）。

## 1. 目标与范围

把已完成的**费用模块**与**退宿模块**打通：办理退宿时，自动把该居住人在住档案下的**未缴账单**结算挂账——记录离场欠费总额到退宿单，并将这些账单置「挂账」状态。退宿流程**不被阻断**（挂账放行）。挂账账单后续仍可缴费结清。仍沿用 H2 内存库 DEMO，子 Agent 驱动 + TDD。

**验收标准：**
- 办理退宿（`CheckoutService.confirm`）在同一事务内完成欠费结算挂账：该档案未缴账单 → 挂账；退宿单回填离场欠费总额；退宿主流程（释放床位/刷新房间/归档档案/置已退宿）不受影响、不被阻断。
- 无欠费时离场欠费金额为 0，账单无变化。
- 挂账账单可后续缴费结清（挂账 → 已缴）。
- 退宿弹窗可预览待结算欠费（张数 + 金额）；退宿单列表展示离场欠费。
- 后端关键逻辑测试覆盖。

## 2. 核心行为与数据流

办理退宿 `CheckoutServiceImpl.confirm`（仍是单一 `@Transactional`）在原有步骤后追加「欠费结算挂账」：

1. 释放床位 + 刷新房间统计 + 归档在住档案 + 退宿单置已退宿（既有逻辑不变）。
2. 调 `FeeBillService.settleArrearsForRecord(record.getId())`：
   - 查该在住档案（checkin_record_id）下所有 **未缴(1)** 账单；
   - 求欠费总额 = Σ amount；
   - 把这些账单状态置 **挂账(4)**；
   - 返回欠费总额（无未缴账单则返回 `BigDecimal.ZERO`）。
3. 退宿单写入 `arrearsAmount` = 返回的欠费总额，保存。

全程同一事务：任一步失败整体回滚（挂账与退宿原子）。

**模块依赖方向：** checkout → fee（confirm 调用 FeeBillService）。fee 不依赖 checkout，无环。

**结算范围：** 以被办理退宿的**在住档案 checkin_record_id** 为准（账单本就按 checkin_record 归集）。

## 3. 数据模型改动

不新增表，只改两处：
- `dms_checkout_order` 加列 `arrears_amount DECIMAL(10,2) DEFAULT 0`（离场欠费金额，办理退宿时回填）。
- 账单状态新增取值 **4 = 挂账**（复用 `dms_fee_bill.status` 列，无 schema 改动）。

种子：现有费用种子保持不变（张三 2026-06 一张未缴账单）；退宿种子单 SEED-CO-1 仍在，办理它即可演示挂账。`arrears_amount` 列加默认 0，旧种子行自动为 0。

## 4. 后端改动

### 4.1 fee 模块
`FeeBillService` 新增：
- `java.util.List<FeeBill> listUnpaidByRecord(Long checkinRecordId)` —— 查该档案未缴(1)账单。
- `java.math.BigDecimal settleArrearsForRecord(Long checkinRecordId)` —— 未缴账单置挂账(4)，返回欠费总额（无则 `BigDecimal.ZERO`）。

`pay` 状态守卫放宽：`status == 1 || status == 4` 才可缴费（挂账也能结清，缴费后 → 已缴(2)）；其余抛 BizException。`voidBill` 维持仅未缴(1)可作废。

新增只读接口 `GET /api/fee/arrears?checkinRecordId=`（JWT）：返回 `ArrearsVO { int count; BigDecimal totalAmount; }`（供退宿弹窗预览，count/total 基于 listUnpaidByRecord）。

### 4.2 checkout 模块
- `CheckoutOrder` 实体加 `private BigDecimal arrearsAmount;`。
- `CheckoutServiceImpl` 注入 `FeeBillService`；`confirm` 末尾 `order.setArrearsAmount(feeBillService.settleArrearsForRecord(record.getId()))`，再 `updateById`。
- `CheckoutOrderVO` 加 `arrearsAmount` 字段（`BeanUtils.copyProperties` 自动带出）。

## 5. 前端改动

- **退宿单页** `views/checkout/order.vue`：
  - 「办理退宿」弹窗打开时调 `getArrears(checkinRecordId)`，显示「待结算欠费：N 张 / ¥X」；有欠费时高亮文案「确认退宿将把欠费挂账」。
  - 列表加一列「离场欠费」展示 `arrearsAmount`（>0 标红，0 显示「-」）。
- **账单页** `views/fee/bill.vue`：`BILL_STATUS` 加 `{ value: 4, label: '挂账', type: 'danger' }`（可筛选）；挂账账单操作列保留「缴费」（作废不可）。
- `api/checkout.ts`：`CheckoutOrder` 类型加 `arrearsAmount?: number`。
- `api/fee.ts`：加 `getArrears(checkinRecordId): Promise<{ count: number; totalAmount: number }>`。

## 6. 测试（TDD + 子 Agent 驱动 + 两阶段审查）

- `settleArrearsForRecord`：多张未缴 → 全部挂账(4) + 总额正确；无未缴 → 返回 0、无变化；已缴/已作废账单不动。
- `listUnpaidByRecord`：只返回未缴。
- `pay` 放宽：挂账(4) 可缴费 → 已缴(2)；已缴/已作废仍拒绝。
- arrears 接口：返回正确 count/totalAmount；鉴权（无 token → 401）。
- `CheckoutService.confirm`：办理退宿后退宿单 `arrearsAmount` = 该档案未缴总额、相关账单变挂账、退宿主流程（床位释放/归档/置已退宿）不受影响；无欠费档案 arrearsAmount=0。
- 全量回归通过。
