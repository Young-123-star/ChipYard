# 账单报表 / 用量趋势 — 设计文档

> 第二阶段第六个子项目。日期 2026-06-17。分支 `feat/report-stats`（从 `feat/utility-billing` 切出——本子项目依赖水电模块的 `dms_meter_reading`，该模块尚未合入 main；需待水电 PR 合并后此分支再并入）。

## 1. 目标与范围

为 DMS 增加**只读统计报表**：对现有 `dms_fee_bill`（账单）+ `dms_meter_reading`（抄表台账）做聚合，提供四块——按账期汇总、按楼栋汇总、欠费排行、水电用量趋势。前端引入 ECharts 出图（趋势折线 + 汇总柱状）。**不新增表、不改现有数据**，纯读。仍沿用 H2 内存库 DEMO，后端 TDD。

**验收标准：**
- 四个只读接口（GET，JWT）各自返回正确聚合数据。
- 口径统一：应收 = 已缴(2) + 未缴(1) + 挂账(4)，**作废(3) 不计**；"未缴"口径含挂账；收缴率 = 已缴 / 应收（应收为 0 → 0）。
- 前端「统计报表」页用 ECharts 呈现账期汇总（柱+收缴率折线双轴）、用量趋势（电/水折线）、楼栋汇总、欠费排行（Top 10 表）。
- 后端关键聚合逻辑测试覆盖；全量回归通过。

## 2. 架构与数据来源

- **无新表**。全部对 `dms_fee_bill` / `dms_meter_reading` 只读聚合。
- **内存聚合**：从 mapper `selectList` 取数后用 Java stream 分组求和（DEMO 数据量小，不写自定义 SQL，便于单测）。
- **模块位置**：`module/fee` 下新增 `ReportService` / `ReportServiceImpl` / `ReportController` + `report` vo 包（报表即费用分析，归 fee 模块，直接用 `FeeBillMapper`/`MeterReadingMapper`；名称回显复用 `RoomService`/`ResidentService`，与现有跨模块读法一致）。
- **金额口径**（统一）：
  - 应收 total = 账单金额合计，排除作废(3)。
  - 已缴 paid = 状态 已缴(2) 的金额合计。
  - 未缴 unpaid = 状态 未缴(1) + 挂账(4) 的金额合计。
  - 收缴率 collectRate = paid / total，保留 2 位（百分数值，如 75.00），total 为 0 时为 0。

## 3. 后端 VO / 接口

均为 GET、JWT、只读，挂 `/api/report`。

### 3.1 按账期汇总 `GET /api/report/period-summary` → `List<PeriodSummaryVO>`（period 升序）
```
PeriodSummaryVO { String period; BigDecimal rentTotal; BigDecimal elecTotal; BigDecimal waterTotal;
                  BigDecimal total; BigDecimal paid; BigDecimal unpaid; BigDecimal collectRate; }
```
- 取全部非作废账单按 period 分组：按 billType 分出住宿(1)=rentTotal / 电(2)=elecTotal / 水(3)=waterTotal；total=三者合计；paid=该期已缴合计；unpaid=该期未缴+挂账合计；collectRate=paid/total。

### 3.2 按楼栋汇总 `GET /api/report/building-summary` → `List<BuildingSummaryVO>`（buildingId 升序）
```
BuildingSummaryVO { Long buildingId; String buildingName; BigDecimal total; BigDecimal paid; BigDecimal unpaid; BigDecimal collectRate; }
```
- 非作废账单 → room_id →（`RoomService.getById` 取 buildingId）→ 按楼栋分组求和；buildingName 由 `BuildingService.getById` 回显。账单无 room_id（理论上不会）则跳过，不影响楼栋汇总。

### 3.3 欠费排行 `GET /api/report/arrears-ranking?limit=10` → `List<ArrearsRankVO>`
```
ArrearsRankVO { Long residentId; String residentName; String employeeNo; BigDecimal unpaidAmount; int unpaidCount; }
```
- 取未缴(1)+挂账(4)账单，按 residentId 分组求欠费总额与张数，按 unpaidAmount 降序取前 limit（默认 10）；居住人名/工号由 `ResidentService` 回显。

### 3.4 水电用量趋势 `GET /api/report/usage-trend` → `List<UsageTrendVO>`（period 升序）
```
UsageTrendVO { String period; BigDecimal electricity; BigDecimal water; }
```
- 取全部抄表台账按 period 分组，meterType=1 的 consumption 合计=electricity，meterType=2 的=water。

## 4. 前端（ECharts + 报表页）

- **依赖**：`dms-frontend` 加 `echarts` + `vue-echarts`；`main.ts` 注册所需组件（LineChart / BarChart + GridComponent / TooltipComponent / LegendComponent / TitleComponent + CanvasRenderer），全局注册 `<v-chart>`（或按页局部引入）。
- **菜单**：新增菜单组「统计报表」，一个页面 `views/report/index.vue`。
- **API/类型**：新增 `api/report.ts`（getPeriodSummary / getBuildingSummary / getArrearsRanking / getUsageTrend）；`types.ts` 加四个 VO 接口。
- **页面布局**（卡片分区）：
  1. **账期汇总**：ECharts 组合图（住宿/电/水柱状 + 收缴率折线，双 Y 轴）+ 明细表（账期/应收/已缴/未缴/收缴率）。
  2. **水电用量趋势**：ECharts 折线图（电、水两条线，X=账期）。
  3. **按楼栋汇总**：横向柱状图或表格（楼栋/应收/已缴/未缴/收缴率）。
  4. **欠费排行**：表格（居住人/工号/欠费额/张数，Top 10）。
- 图表配色复用蓝色主题色板（`--dms-accent` 等），与全局风格一致。

## 5. 测试

- ReportController：四接口鉴权（无 token → 401）。
