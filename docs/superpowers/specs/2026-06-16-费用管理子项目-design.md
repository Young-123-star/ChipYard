# 费用管理子项目 — 设计文档（住宿费账单闭环）

> 第二阶段第三个业务子项目。日期 2026-06-16。分支 `feat/fee-module`（从合并了退宿的最新 `main` 切出）。

## 1. 目标与范围

为 DMS 增加**住宿费账单闭环**：收费标准（按房型月单价）→ 按月生成账单 → 缴费登记 → 欠费查询。沿用项目一贯的 DEMO 形态（H2 内存库 + 种子数据，零外部依赖启动），子 Agent 驱动 + TDD 实现。

**验收标准：**
- 后端零外部依赖启动（H2 + 种子）。
- 管理员可维护收费标准（房型 → 月单价，房型唯一）。
- 管理员选账期一键生成账单：对所有在住档案各生成一张该月账单，金额取其房间房型的月单价快照；同档案同月幂等不重复；无对应收费标准的档案跳过并回报数量。
- 缴费：未缴 → 已缴 + 记缴费时间/方式；非未缴拒绝。
- 作废：账单可作废。
- 欠费查询：账单列表按"未缴"筛选即得。
- 后端关键逻辑测试覆盖。

## 2. 数据模型

新增 2 张表，沿用 `BaseEntity`（created_at / updated_at / deleted_at 逻辑删除）。

### dms_fee_standard（收费标准）
| 列 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| room_type | TINYINT | 房型（对齐 `dms_room.room_type`），业务唯一 |
| monthly_price | DECIMAL(10,2) | 月单价（元） |
| remark | VARCHAR(200) | 备注，可空 |
| created_at / updated_at / deleted_at | DATETIME | 审计 + 逻辑删除 |

### dms_fee_bill（住宿费账单）
| 列 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| bill_no | VARCHAR(64) | 账单号 `BILL-{recordId}-{YYYYMM}`，幂等键 |
| checkin_record_id | BIGINT | 关联在住档案（价格按其房间房型取） |
| resident_id | BIGINT | 冗余快照，便于查询/展示 |
| room_id | BIGINT | 冗余快照（生成时） |
| period | VARCHAR(7) | 账期 `YYYY-MM` |
| amount | DECIMAL(10,2) | 金额 = 生成时房型月单价快照 |
| status | TINYINT | 1 未缴 / 2 已缴 / 3 作废 |
| paid_at | DATETIME | 缴费时间，可空 |
| pay_method | TINYINT | 1 现金 / 2 转账，可空 |
| created_at / updated_at / deleted_at | DATETIME | 审计 + 逻辑删除 |

**关键决定：**
- 账单**关联在住档案 `checkin_record_id`**（不是直接关联居住人），因价格取决于其房间房型；`resident_id`/`room_id` 为生成时冗余快照，便于展示且不受日后换房影响。
- `amount` 是**生成那一刻的房型单价快照**——日后修改收费标准不影响已生成账单。
- 幂等键：`(checkin_record_id, period)` 逻辑唯一（应用层 select-before-insert 保证；DEMO 不加 DB 唯一约束，与既有模块一致）。
- 金额用 `DECIMAL`（不用 float）。

## 3. 后端模块结构

新增 `fee` 模块（对称 `checkout`/`checkin` 模块）：
- `entity/`：FeeStandard、FeeBill
- `mapper/`：FeeStandardMapper、FeeBillMapper
- `dto/`：FeeStandardDTO、BillQuery（分页 + period/status/residentId）、GenerateBillsDTO（period）
- `vo/`：FeeBillVO（账单字段 + residentName/employeeNo/roomNumber/roomType）、GenerateResultVO（generated/skipped）
- `service/`：FeeService + FeeServiceImpl
- `controller/`：FeeController

费用是内部业务，**无 OA/HCP 对接**。

## 4. 接口（均需 JWT）

- 收费标准 CRUD：`GET/POST/PUT/DELETE /api/fee/standards`（POST/PUT 按 room_type 查重）
- 账单分页列表：`GET /api/fee/bills`（分页 + period/status/residentId 筛选 + 居住人/房间回显）
- 生成账单：`POST /api/fee/bills/generate { "period": "2026-07" }`
- 缴费：`POST /api/fee/bills/{id}/pay { "payMethod": 1 }`
- 作废：`POST /api/fee/bills/{id}/void`

统一返回 `R<T>`；分页返回 `PageResult<T>`；service 抛 `BizException`。

## 5. 核心流程

### 5.1 生成账单（`@Transactional`，幂等批量）
1. 入参 period（`YYYY-MM`）。
2. 查所有在住档案（`dms_checkin_record.status = 1`）。
3. 逐条：按 record → room → room_type 查收费标准。
   - 标准缺失 → 跳过，`skipped++`。
   - `(checkin_record_id, period)` 已存在账单 → 跳过（幂等，不计入 generated）。
   - 否则建账单：bill_no=`BILL-{recordId}-{YYYYMM}`，amount=标准月单价快照，status=未缴，冗余 resident_id/room_id。`generated++`。
4. 返回 `GenerateResultVO { generated, skipped }`。

> 管理员补齐收费标准后可再次对同一账期生成，已生成的不会重复，只补齐之前跳过的。

### 5.2 缴费
- 仅 `未缴(1)` 可缴 → 置 `已缴(2)` + paid_at=now + pay_method。
- 非未缴 → 抛 BizException。

### 5.3 作废
- 仅 `未缴(1)` 可作废 → 置 `作废(3)`；已缴 / 已作废不可作废 → 抛 BizException（与缴费的状态守卫一致）。

### 5.4 欠费查询
- 复用账单列表 `status=未缴` 筛选，不单独建接口。

## 6. 前端

新增菜单组「**费用管理**」（与「资源管理」「入住管理」并列）：

1. **收费标准** `views/fee/standard.vue`：表格（房型/月单价/备注）+ 新增/编辑/删除弹窗，房型用现有字典渲染。
2. **住宿费账单** `views/fee/bill.vue`：
   - 筛选条：账期（month picker）/ 状态 / 居住人；汇总条（共 X 张 · 已缴 · 未缴 · 金额合计，覆盖全量）。
   - 「生成账单」按钮 → 选账期 → generate → toast「生成 N 张 / 跳过 M 张」。
   - 表格：账单号 / 居住人(姓名+工号) / 房间 / 账期 / 金额 / 状态 tag / 缴费时间；操作列未缴 → 「缴费」(选现金/转账) + 「作废」。
- 新增 `api/fee.ts`；`api/types.ts` 加 FeeStandard/FeeBill；`utils/dict.ts` 加 BILL_STATUS / PAY_METHOD / ROOM_TYPE（若无）。
- 路由 + layout 菜单组加两项。
