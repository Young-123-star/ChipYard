# 维修工单管理 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an admin-only repair order module for manual room repair requests, acceptance, completion, cancellation, and filtered listing.

**Architecture:** Follow existing package-by-feature modules. Add one `dms_repair_order` table and one `repair` backend package; the service owns state guards and uses existing `RoomService`, `BuildingService`, and `ResidentService` for validation and display names. Frontend adds one table/form page using existing Element Plus, request wrapper, router, menu, types, and dictionaries.

**Tech Stack:** Spring Boot 3.2, MyBatis-Plus, H2, JUnit/Spring MockMvc, Vue3, Vite, Element Plus, Axios.
