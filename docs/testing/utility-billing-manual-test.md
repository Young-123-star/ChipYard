# 水电计算模块手工验收指南

> 仅限独立测试数据库。两个数据脚本都会清空宿舍业务数据，严禁在生产或共享数据库执行。

## 1. 环境与数据

1. 切换到 `codex/utility-billing`，启动后端和前端。
2. 本地登录账号：`admin / admin123`。
3. 打开 `http://localhost:8080/h2-console`：
   - JDBC URL：`jdbc:h2:mem:dms`
   - User Name：`sa`
   - Password：留空
4. 冒烟测试执行 [utility-billing-smoke.sql](./utility-billing-smoke.sql)。
5. 完整矩阵执行 [utility-billing-matrix.sql](./utility-billing-matrix.sql)。
6. 固定账期：`2026-08`；周期应显示 `2026-07-25 至 2026-08-24`。
7. 固定电价：`0.5383`；水价：`4.1500`。

恢复基线：重启后端会重建本地内存数据库；也可以重新执行相应 SQL。每个异常用例执行前必须恢复基线。

## 2. 30 分钟冒烟测试

执行 `utility-billing-smoke.sql` 后：

1. 进入“房间管理”，编辑 `T36-01`：
   - 结算方式：房间级
   - 账户编码：留空
   - 用电规则：房间 250 度
   - 用水规则：房间 17 吨
2. 保存并确认：
   - 房间账户编码自动变为 `T36-01`。
   - 楼栋页面水电配置为 `1/1`。
   - “抄表/水电 → 结算账户”显示“可结算”。
3. 在“抄表台账”选择 `2026-08`，依次录入：
   - 电表：上期 100，本期 400，用量 300。
   - 冷水表：上期 10，本期 22，用量 12。
   - 热水表：上期 5，本期 13，用量 8。
4. 在“月度结算”点击“结算预览”，确认：
   - 实际费用 `244.49`
   - 员工承担 `39.37`
   - 公司承担 `205.12`
5. 点击“确认生成”，确认：
   - 1 条有效结算。
   - 电费账单 `26.92`，水费账单 `12.45`。
   - 账单类型分别显示“电费”“水费”，不能显示数字 `3`。
6. 生成后再次录入同账户抄表，应提示读数已锁定；再次生成应被阻止。
7. 未缴费时作废：
   - 结算和个人账单都变为“已作废”。
   - 修改读数后可重新生成。
8. 编辑该房间后点击“新建房间”，新建表单的水电配置必须为空，不能继承 `T36-01`。

## 4. 异常与联动

每个用例先恢复矩阵基线，只改变一项：

| 编号 | 操作 | 预期 |
|---|---|---|
| E01 | 删除任一必需抄表后预览 | 显示缺表，整月不能生成 |
| E02 | 将 H124 户总电量改为 500，分表仍为 260/340 | 提示公共用电为负，不能生成 |
| E03 | 夫妻房只保留 1 人或增加到 3 人 | 提示截止日人数异常，不能生成 |
| E04 | 同一户两个房间设置不同规则 | 账户显示“需修正”，不能生成 |
| E05 | 用电、用水都设为“不计” | 账户显示“需修正”，不能生成 |
| E06 | 首次抄表不填上期数 | 保存失败 |
| E07 | 本期数小于上期数 | 保存失败 |
| E08 | 任一账户错误，其余账户正确 | 不得出现部分结算 |
| E09 | 生成后修改读数或重复生成 | 操作被阻止 |
| E10 | 未缴结算作废并重算 | 原结算/账单作废，新结算有效 |
| E11 | 缴纳任一关联账单后作废结算 | 作废失败，已缴数据不变 |
| E12 | 24 日入住、25 日入住、24 日退宿 | 仅 24 日入住人员计费 |
| E13 | 修改单价后查看历史结算 | 历史金额不变，新账期使用新价格 |
| E14 | 检查楼栋、导入、账单、退宿欠费和报表 | 配置数、归组、金额及历史数据正常 |
