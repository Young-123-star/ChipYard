# RuoYi-Vue3 可借鉴点记录

## 项目来源

- GitHub: [Young-123-star/RuoYi-Vue3](https://github.com/Young-123-star/RuoYi-Vue3)
- 定位：Vue3 + Element Plus + Vite 的后台管理系统前端模板，配套 RuoYi 后端生态。
- 使用方式：作为后台通用能力设计参考，不直接整体迁移，不引入未评估依赖。

## 可借鉴方向

- 角色 / 菜单 / 按钮权限 RBAC：适合补齐本系统当前只有登录鉴权、缺少角色菜单权限的问题。
- 权限指令 `hasPermi` / `hasRole`：适合改造成 Vue3/TypeScript 版本，用于按钮级权限控制。
- 字典管理与 `DictTag` 展示模式：适合替换当前前端 `dict.ts` 中硬编码的房间状态、床位状态、维修状态、费用状态、性别、人员类型等字典。
- 用户、部门、岗位等后台基础管理模型：适合后续做系统用户管理和组织权限时参考。
- 操作日志 / 登录日志：适合上线后记录导入、入住、退宿、费用、维修等关键操作，便于审计和排查。
- Excel 导入弹窗、通用下载工具：适合优化现有数据初始化导入体验，并支撑台账导出。
- 文件上传、图片上传、图片预览：适合后续维修工单附件、房间照片、处理前后照片等场景。
- 分页、右侧工具栏、面包屑、TagsView 等通用组件：可按需借鉴，不作为近期主线。

## 不建议直接搬用

- 整套 Layout / 动态路由：当前系统已有蓝色主题和固定业务菜单，整套替换成本高。
- 代码生成器：当前业务模型已形成，短期收益低，容易引入复杂度。
- 定时任务：暂时没有明确业务场景。
- 服务监控、缓存监控、Druid 监控：依赖后端配套，当前 Docker 日志和 Spring Boot 监控需求还不强。
- 在线表单构建器：与当前宿舍管理业务主线关系弱，暂不引入。

## 建议优先级

1. 字典管理
2. 台账导出 / 通用下载
3. 权限角色 RBAC
4. 维修附件上传
5. 操作日志 / 登录日志

## 后续设计原则

- 以后做相关功能方案时，先查本文件，再决定是否参考 RuoYi 的页面结构、接口命名和组件模式。
- 以当前 ChipYard 架构为主，只吸收成熟后台模式，不复制无关模块。
- 每个借鉴点都应按本系统业务重新设计接口、数据表和权限边界。
