# 水电费模块重设计方案

> 状态：已确认，暂不开发
> 唯一业务依据：`docs/颀园水电费计算方式.xlsx` 的“水电费计算说明”页签

## 1. 计算规则

- 账期：上月 25 日至本月 24 日。
- 电价：0.5383 元/度，以 250 度为补贴标准；135 元仅为近似说明。
- 水价：4.1500 元/吨。
- 超标量统一使用 `max(实际用量 - 补贴量, 0)`。
- 员工承担先计算房间或户的员工承担总额，再按本期计算住宿人数分摊。
- 公司承担额 = 实际总费用 - 员工承担总额。
- 金额按分 `HALF_UP`，尾差并入房间号、入住档案 ID 最小的记录。
- 本期计算住宿人数取账期截止日（24 日）的在住人员。

### 1.1 1、2、4 号楼

电费适用于 4/6/8 人间：

```text
户总用电量 = 户电总表本月数 - 上月数
房间用电量 = 房间电表本月数 - 上月数
公共用电量 = 户总用电量 - 所有房间用电量合计
房间实际用电量 = 房间用电量 + 公共用电量 / 户内房间数
房间员工承担总额 = max(房间实际用电量 - 250, 0) * 0.5383
个人承担额 = 房间员工承担总额 / 本期计算住宿人数
户公司承担额 = 户总电费 - 所有房间员工承担总额合计
```

水费适用于一期全部房型：

```text
户总用水量 = 冷水用量 + 热水用量
户员工承担总额 = max(户总用水量 - 50, 0) * 4.15
房间员工承担额 = 户员工承担总额 / 户内房间数
个人承担额 = 房间员工承担额 / 本期计算住宿人数
户公司承担额 = 户总水费 - 户员工承担总额
```

### 1.2 3、6 号楼

适用于 4 人间：

```text
房间电员工承担总额 = max(房间用电量 - 250, 0) * 0.5383
房间水员工承担总额 = max(冷水用量 + 热水用量 - 17, 0) * 4.15
个人承担额 = 员工承担总额 / 本期计算住宿人数
公司承担额 = 房间实际总费用 - 员工承担总额
```

### 1.3 5 号楼

- 夫妻房：水电按实际费用结算，必须有 2 名截止日在住人员，每人承担 50%；人数不为 2 时阻止结算。
- 主管房：电 250 度、水 17 吨以内由公司承担，超出部分由在住人员分摊。
- 标准间：水电实际费用全部由公司承担，不生成员工账单。
- 普通空房不生成个人账单，费用全部计入公司承担。

## 2. 水电结算账户与房间数据

采用半自动创建水电结算账户。房间数据增加：

- `settlementMode`：户级或房间级。
- `utilityAccountCode`：水电结算户号。

创建规则：

- 房间级：保存房间时自动创建“一房一账户”，结算户号自动使用房间号。
- 户级：结算户号必填；同一楼栋、相同结算户号的房间自动绑定同一账户。
- 批量导入房间时，按“楼栋 + 结算户号”自动归组。
- 不根据房间号、床位数或在住人员猜测户级关系。
- 房型字典补充 6 人间、8 人间。
- 楼栋表不增加水电规则字段，只展示水电配置完成度。

每个结算账户分别保存：

- `electricityRule`：户级 250 度、房间 250 度、夫妻房各半、公司承担、不适用。
- `waterRule`：户级 50 吨、房间 17 吨、夫妻房各半、公司承担、不适用。

户级规则的账户可以绑定多个房间；房间级及 5 号楼规则的账户只能绑定一个房间。

## 3. 抄表与结算

抄表记录包含账户、抄表对象、表类型、账期、上期读数、本期读数和用量。

- 户级电费：户电总表和每个房间电分表。
- 户级水费：户冷水表和热水表。
- 房间级：房间电表、冷水表和热水表。
- 第一条记录录入上期和本期读数，后续自动取得上期数。
- 本期读数不得小于上期读数。
- 已结算读数禁止修改。

新增结算主表和房间结果表，保存规则及单价快照、实际用量、补贴量、超标量、实际费用、员工承担、公司承担、结算版本和状态。

员工承担继续生成现有电费、水费账单；公司承担只进入结算明细。

页面分为结算配置、抄表录入、账期结算三个页签。

生成前必须预览并校验缺表、负用量、户总表小于分表合计、房间未归组、规则缺失和夫妻房人数异常。存在错误时整月不生成。

未缴结算可作废并重新生成修订版；存在已缴账单时禁止作废。

- `GET /api/fee/utility-settlements`
- `GET /api/fee/utility-settlements/{id}`
- `POST /api/fee/utility-settlements/{id}/void`

迁移：

- 使用新的 Flyway 迁移增加账户、房间关联、结算及结果表。
- 单价字段调整为 4 位小数。
- 同步更新本地 H2 初始化脚本。
- 旧账单保持不变，旧抄表记录只读保留。
- 不导入或校验 Excel 其他页签中的楼层、户数、房间数、床位数等信息。

测试覆盖四类计费方式、公共用电守恒、补贴边界、空房、夫妻房人数、分钱尾差、缺表校验、重复生成、结算锁定、作废重算及现有缴费、挂账流程回归。
